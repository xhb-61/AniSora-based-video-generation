# AniSora 视频生成模型部署与推理优化项目报告

## 1. 项目名称

基于 Linux 服务器的 AniSora 视频生成模型部署、兼容改造与推理实验

## 2. 项目背景

围绕 Bilibili AniSora 视频生成模型，完成从模型获取、环境搭建、依赖适配、推理配置设计，到多组 demo 运行与效果对比分析的完整落地流程。项目重点不在训练，而在于将一个原本偏 Windows 运行时的低显存推理包迁移到 Linux 多卡 GPU 服务器，并验证不同分辨率、步数、时长配置对生成质量和资源消耗的影响。

## 3. 个人职责

- 在远程 GPU 服务器上完成 AniSora 相关仓库、模型与推理环境搭建
- 分析 12GB VRAM 版本推理包结构，定位 Linux 环境下的运行阻塞点
- 针对 Windows 专用扩展依赖进行 Linux 兼容改造，打通最小可运行链路
- 设计并执行多组推理实验，包括单样例 demo 与批量 `inference_360.txt` 样例
- 对比 3s 与 6s、低配 smoke 与 480p/4 steps 配置的生成效果差异
- 沉淀可复用配置、脚本思路与实验结论，为后续演示或进一步优化提供基础

## 4. 技术环境

- 服务器：8 x RTX 4090
- 操作系统：Linux
- Python 环境：Conda Python 3.10
- 深度学习框架：PyTorch 2.11.0 + CUDA 13.0
- 模型版本：AniSora 12GB 推理包，基于 `wan2.1` 架构的 FP8 量化权重
- 关键推理组件：
  - diffusion / DiT：`models/anisora/fp8`
  - VAE：`Wan2.1_VAE.pth`
  - T5：`models_t5_umt5-xxl-enc-bf16.pth`
  - CLIP：`clip-fp8.pth`

## 5. 项目难点与解决方案

### 5.1 低显存推理包与 Linux 运行环境不兼容

AniSora 12GB 版本自带的运行时包含 Windows `.pyd` 扩展，直接在 Linux 服务器上无法使用，核心阻塞点包括：

- `q8_kernels_cuda`
- `vllm` 自定义量化算子
- `sageattention` 等加速依赖缺失

### 5.2 解决方案

为尽快验证模型可运行性，采用“最小可运行优先”的思路，完成以下兼容改造：

- 基于 LightX2V 的推理代码结构，定位量化线性层与算子调用路径
- 编写 Linux 侧的 `q8_kernels` / `vllm._custom_ops` 兼容 shim，使用纯 PyTorch fallback 替代缺失的 Windows 专用扩展
- 调整配置使 CLIP 量化路径改走 `fp8-q8f` 分支，避开不可用的 `cutlass_scaled_mm`
- 保留 `cpu_offload`、`lazy_load`、`offload_to_disk` 等低显存策略，降低显存压力
- 单独维护 Linux 可运行配置文件，避免破坏原始推理包内容

## 6. 实验设计

### 6.1 单样例 demo

使用单张输入图与人工编写 prompt，验证以下配置：

- smoke 配置：约 320p，`infer_steps=1`，`49` 帧，约 `3.06s`
- 质量配置：480p，`infer_steps=4`，`49` 帧，约 `3.06s`
- 长时长配置：480p，`infer_steps=4`，`97` 帧，约 `6.06s`

### 6.2 批量样例实验

基于 `inference_360.txt` 中 3 条角色旋转样例，分别运行：

- 480p / 4 steps / 49 帧
- 480p / 4 steps / 97 帧

用于观察角色转身、服饰保持、后半段漂移等现象。

## 7. 关键实验结果

### 7.1 运行打通结果

- 成功在 Linux 服务器上跑通 AniSora 12GB 版本推理
- 支持单样例生成与批量样例生成
- 成功生成 3s 与 6s 两类视频结果

### 7.2 资源与时长表现

- 480p / 4 steps / 49 帧：可稳定生成约 `3.06s` 视频
- 480p / 4 steps / 97 帧：可稳定生成约 `6.06s` 视频
- `97` 帧版本在 RTX 4090 上实测显存约提升到 `17GB - 18GB` 左右

### 7.3 画质与稳定性观察

- 从 smoke 配置提升到 480p / 4 steps 后，人物结构、服饰纹理、边缘清晰度均明显改善
- 从 3s 扩展到 6s 后，动作完整度明显提升，角色可完成更充分的转身或转圈
- 但时长增加后，个别样例在后半段会出现一定设定漂移，尤其是服装细节和下装形态变化更明显
- 综合来看：
  - 样例 1 和样例 3 在 6s 配置下表现较稳
  - 样例 2 在 6s 配置下动作更完整，但后半段一致性相对更弱

## 8. 项目产出

- Linux 可运行配置文件
- 单样例与批量样例推理结果
- 3s / 6s、低配 / 高配实验对比结果
- 一套可复用的 AniSora 推理实验流程

## 9. 项目价值总结

该项目的价值主要体现在三方面：

- 打通了一个原本偏 Windows 运行时的低显存视频生成包在 Linux GPU 服务器上的可运行链路
- 通过兼容改造与配置设计，验证了量化权重、offload 与 lazy load 组合在视频生成推理中的实际可用性
- 通过多组实验，明确了分辨率、步数、时长对生成质量与稳定性的影响，为后续进一步优化 prompt、推理参数和模型版本选择提供依据
